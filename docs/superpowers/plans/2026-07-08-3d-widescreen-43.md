# 3D Widescreen-From-4:3 (No Parallax) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The 3D diorama presents the 4:3-corrected 320x200 stage centred in a window of any shape, filling it edge-to-edge with a mirrored wall extension; the parallax camera is removed.

**Architecture:** A new pure helper `scene_frustum(win_w, win_h)` in `src/video3d/scene3d.cpp` (bumpy_core, unit-testable) picks the frustum aspect and half-height so the field is always whole. `SceneRenderer::render` uses it with a fixed camera; the wall quad is sized to the frustum at wall depth with centre-anchored UVs over a `GL_MIRRORED_REPEAT` texture. `sdl_app.cpp` passes the full window as the viewport and drops the camera-easing state.

**Tech Stack:** C++20, OpenGL 3.3 core, SDL3, Catch2 v3 (tests), CMake preset `windows-debug` (VS generator).

**Spec:** `docs/superpowers/specs/2026-07-08-3d-widescreen-43-design.md`

## Global Constraints

- CRT correction factor is exactly `240/200 = 1.2` (constant `kCrtPixelAspect`).
- The 320x200 field must be fully visible at every window shape — never cropped, never letterboxed; extra window area shows mirrored wall.
- Camera distance stays `scene_camera_distance()` at every window shape (perspective character never changes).
- The flat 2D path (menus, map, 2D gameplay, Alt+A behaviour there) is untouched.
- Build: `cmake --build build/windows-debug --config Debug`; tests run from repo root: `./build/windows-debug/Debug/bumpy_tests.exe`.

```cpp
// Frustum shape that fills a window of any aspect with the 4:3-corrected stage:
// `aspect` is the world-unit width/height ratio for mat4_perspective and
// `half_height` the board px visible above/below centre at z=0. The 320x200
// field is always whole: windows wider than 4:3 pin half_height at 100 and
// widen, narrower windows grow half_height so all 320 px of width stay visible.
// Camera distance is unchanged either way (only the fov widens).
struct SceneFrustum {
    float aspect{};
    float half_height{};
};
[[nodiscard]] SceneFrustum scene_frustum(int window_w, int window_h);
```

- [ ] **Step 4: Implement in `src/video3d/scene3d.cpp`**

Insert after `scene_camera_distance()` (file already includes `<cmath>`; `std::max` needs `#include <algorithm>` — add it to the include block):

```cpp
SceneFrustum scene_frustum(int window_w, int window_h) {
    if (window_w <= 0 || window_h <= 0) {
        return {320.0f / 200.0f, 100.0f};  // degenerate: exact-field default
    }
    const float aspect =
        kCrtPixelAspect * static_cast<float>(window_w) / static_cast<float>(window_h);
    return {aspect, std::max(100.0f, 160.0f / aspect)};
}
```

```cpp
    const float dist = scene_camera_distance();
    // 4:3-corrected stage in any window: world px present kCrtPixelAspect taller
    // than wide (the flat path's Alt+A 4:3 look). The camera never moves; the fov
    // only grows when the window is narrower than 4:3, so the field is always
    // whole and centred and wider windows just reveal more wall at the sides.
    const SceneFrustum fr = scene_frustum(vp.w, vp.h);
    const float fovy = 2.0f * std::atan(fr.half_height / dist);
    const Mat4 proj = mat4_perspective(fovy, fr.aspect, 1.0f, 2000.0f);
    const Mat4 view = mat4_translate(0.0f, 0.0f, -dist);
    const Mat4 mvp = mat4_multiply(proj, view);
    const float light_gx = light_x - 160.0f;
    const float light_gy = 100.0f - light_y;
```

- [ ] **Step 3: `scene_renderer.cpp` — wall quad sized to the frustum, mirrored UVs**

```cpp
    // texels exactly behind the field (1 texel <-> 1 stage px on screen) and the
    // GL_MIRRORED_REPEAT wrap continues it past the edges. Drawn without depth so
    // everything else stacks in front.
    {
        const float cover = (dist - kWallZ) / dist;
        const float sx = fr.half_height * fr.aspect * cover + 2.0f;
        const float sy = fr.half_height * cover + 2.0f;
        const float u0 = 0.5f - sx / (320.0f * cover);
        const float u1 = 0.5f + sx / (320.0f * cover);
        const float v0 = 0.5f - sy / (200.0f * cover);
        const float v1 = 0.5f + sy / (200.0f * cover);
        QuadFace wall;
        wall.corners = {-sx, sy, kWallZ, sx, sy, kWallZ, sx, -sy, kWallZ, -sx, -sy, kWallZ};
        wall.uv = {u0, v0, u1, v0, u1, v1, u0, v1};
```
